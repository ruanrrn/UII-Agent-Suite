## 1. 定位与核心任务

这是「联影智能 AI 能力平台」架构图中**门户（Portal）**模块的设计。在完整架构里，门户与右侧 **MCP Server** 配对：门户承载 *登录·购买·续费、服务目录、下载 Skill/MCP*，MCP Server 把私域专用 AI 能力按标准协议输出给智能体。

**本阶段的核心任务不是做一个可日常运维的系统，而是：向外界（医院）展示整套理念的概念构想，把"人和智能体共用一个 AI 能力门户"这件事讲清楚、讲到位、并让对方能"摸到"。** 网站尚未进入日常使用。

因此本门户定性为 **A+ · 概念展示型门户**：以**发现型目录的适当"营销感"**为主体，叠加**理念叙事层**，并提供一个**可点击的伪控制台 demo**让医院感知登录后的样子。

---

## 2. 目标与非目标

### 目标
- 用一条可滚动的叙事，把平台理念（双受众 / 私域安全 / MCP 标准输出 / 商业化闭环）讲成医院听得懂的人话。
- 以精致的能力目录展示影像 AI 能力（**全部为联影 FDA 已获批的真实 AI 软件**）。
- 清晰表达**人机双轨**：同一能力，人怎么发现评估、智能体怎么经 MCP 连接。
- 提供可点击的**伪控制台 demo**，让医院直观感受"开通→连接→使用"后的形态。
- 用医疗级的信任/合规表达（FDA 510(k) 等真实认证）建立可信度。

### 非目标（YAGNI，本期明确不做）
- ❌ 不做真实登录 / 鉴权 / 多租户后端。
- ❌ 不做真实支付 / 订单 / 计量计费后端。
- ❌ 不实现真实 MCP Server / 真实能力调用。
- ❌ 不做厚重的运维控制台（仅以伪控制台 demo 呈现概念）。
- ❌ **不虚构能力**：目录条目必须可查到真实 FDA 认证信息（K 号）。

> 说明：双轨、中央多租户 SaaS、私域部署等决策**依然成立**，但在本期身份是"**要讲给医院的卖点/概念**"，而非"要实现的功能"。

## 3. 用户与场景

| 角色 | 优先级 | 在本站的主要诉求 |
|---|---|---|
| **院方 IT / 平台管理员** | 英雄用户 | 理解平台价值、登录后会得到怎样的控制台、如何接入 |
| 临床 / 影像科决策者 | 次要 | 评估临床价值、对比能力、判断是否引入 |

整体语气：**可信、克制、有 AI 科技感**。

## 5. 参考与灵感（4 类原型）

| 原型 | 代表 | 借鉴到本门户 |
|---|---|---|
| A 模型/能力中心 | Hugging Face、Replicate、通义 Models（v1 底子） | 发现层卡片网格 + 详情 + "Use this"代码片段 |
| **B MCP 注册表（最近）** | 官方 MCP Registry、Smithery、mcp.so、Glama | **门户=人面向注册表 / MCP Server=协议端点**；连接配置复制；**验证徽章** |
| C API 市场（交易） | RapidAPI、AWS/Azure 模型目录、Vertex Model Garden | 开通/订阅/凭据/用量的**概念**表达 |
| D 文档即产品（双轨内容） | Stripe Docs + `llms.txt`、Vercel AI SDK | `llms.txt` + "把本页复制成 markdown 喂给你的 AI"；打磨水准 |

**合成判断**：本门户 ≈ B（注册表结构与信任）+ C（交易/租户概念），外壳取 D（打磨）+ A/v1（卡片语言），叠加**医疗 FDA 认证的真实合规表达**作为差异化护城河。

**与 v1 的关系**：发现层演进 v1 的"卡片网格 + 详情"DNA；v1 的"复制这段话粘贴给 AI 助手"是双轨桥接雏形，本期工业化为 `llms.txt` / 连接配置 / 机器视图。

## 6. 品牌与视觉系统

### 6.1 颜色

**关键色（唯一）**
- 青绿 `#0fd78a` —— 专指 **AI / 智能体 / 连接在线 / 主 CTA / 强调**。深色底直接用；浅色底文字用加深版 `#0a8f5d` 保对比；浅底色块 `#e6faf2` / `rgba(15,215,138,.12)`。

**中性基底**
- 黑 `#121212`（hero / 顶栏 / 深色叙事区 / 控制台外壳）
- 近黑 `#0d1117`（机器视图 / 代码 / 连接配置块）
- 白 `#ffffff`（内容区 / 目录 / 详情正文）
- 灰阶（沿用 v1）：`#fafbfc` `#f3f4f6` `#e5e7eb` `#9ca3af` `#6b7280` `#4b5563` `#111827`

**装饰色（非关键 · 允许极少量使用）**
- 金 `#c59d62`（集团）—— 仅「企业版 / 认证」等极少量高端点缀。
- 青蓝 `#0599c8`（集团）—— 可用于次级链接等极少量场景。
- 青绿仍是唯一**关键**色，主导强调与 AI 语义。

### 6.2 字体（沿用 v1）
- Display：`Space Grotesk`；Body：`Inter` + 中文 `PingFang SC` / `Microsoft YaHei`；Mono：`JetBrains Mono`。

### 6.3 美学策略：明暗分区
- **深色（黑 + 青绿）**：Hero、理念叙事、如何工作、机器视图、控制台外壳 —— AI 科技感。
- **浅色（白 + 灰 + 青绿点）**：能力目录、能力详情正文 —— 可读性。
- 圆角 / 阴影 token 沿用 v1。

### 6.4 组件清单
顶栏（深色 + 语言切换）· Hero（深色）· 能力卡片（浅色，演进 v1）· 类型/筛选侧栏 · 能力详情（人/机双栏）· **连接配置块**（近黑、mono、可复制）· **FDA 认证徽章** · 伪控制台外壳（深色导航 + 浅色内容）· 机器视图产物芯片（`/llms.txt`、目录 API、`/mcp`）· CTA · 页脚。

### 6.5 国际化（中英双语）
- 全站 **zh-CN / en** 双语；顶栏提供语言切换。
- 内容模型每个文案字段提供 `zh` / `en` 两份（见 §9.1）；产品名（uAI…）保持英文原名。
- `/llms.txt` 提供中英两版或双语合并版，便于人/机两侧都可读。

## 8. 页面级要素

### 1. 首页 `/`
- **Hero**：主张——"把院内影像 AI，变成人和智能体都能直接调用的标准服务"；主 CTA「浏览能力」+ 次 CTA「体验控制台」；深色 + 青绿。
- **理念「它是什么」**：图示化分层——双受众 / 私域安全 / MCP 标准输出 / 商业化闭环。
- **能力目录预览**：精选 3–6 个能力卡片 + "查看全部"。
- **如何工作（缩略）**：发现→开通→连接 三步，链到 `/how-it-works`。
- **信任/认证带**：FDA 510(k) Cleared 等真实认证徽章。
- **页脚 CTA**：体验控制台 / 浏览能力 / GitHub。（**不含"预约演示"**）

### 2. 能力目录 `/catalog`
- 顶部：搜索 + 排序 + 视图切换（沿用 v1）。
- 左侧：类型筛选（临床 AI 辅助 / AI 影像平台·后处理 / AI 重建增强 / Skill）+ 模态（CT / MR / PET / X 线）。
- 主区：能力卡片网格。卡片字段见 §9。

### 3. 能力详情 `/capability/:id`
- 头部：图标 + 名称 + ID（可复制）+ 类型/模态徽章 + **FDA 认证徽章（K 号）**。
- 价值叙事：临床用途、输入/输出、适用场景（双语）。
- **人机双轨双栏**：
  - 人类视图：临床说明、评估要点、（概念）开通入口。
  - 机器视图（近黑块）：MCP 连接配置示例、`llms.txt` 链接、目录 API 片段、"复制成 markdown"。
- 关联链接：FDA 510(k) 摘要（accessdata）/ GitHub。

### 4. 如何工作 `/how-it-works`
- 分步/滚动叙事：① 人在门户发现并开通 → ② 拿到凭据/连接配置 → ③ 智能体（AI PACS）经 MCP 连接 → ④ 私域内调用、数据不出院。

### 5–8. 伪控制台 demo（假数据驱动）
- **统一外壳**：深色左导航 + 浅色内容区；顶部标注「DEMO · 演示数据」。
- 控制台首页 / 我的服务 / 连接与凭据（智能体落点）/ 用量·订阅，均用预置假数据渲染。

## 10. 人机双轨机制（概念）

| 环节 | 人类视图 | 机器/智能体视图 |
|---|---|---|
| 发现 | 目录卡片、搜索筛选、详情叙事 | `/catalog` 目录 API（静态 JSON）、`/llms.txt` |
| 评估 | 临床价值、FDA 认证徽章 | 能力 metadata、"复制成 markdown" |
| 开通 | 概念入口 / 伪控制台 | （概念）凭据下发 |
| 连接 | 连接与凭据页展示配置 | MCP 连接端点示意、API Key |

**本期落地原则**：`llms.txt` 与 `catalog.json` **作为真实静态文件存在**；MCP 端点 / 凭据下发为**概念示意**。

## 11. 技术方案

- **静态站点，无真实后端。** 沿用 v1 的 **vanilla HTML/CSS/JS**（零依赖、易托管）；多页面用独立 HTML，共享 `styles.css` 与数据 JS。
- **中英双语**：内容数据按 `{zh,en}` 字段组织；轻量 i18n（语言切换 + 文案字典），无需框架。
- 伪控制台用**预置 JSON 假数据**渲染。
- 机器视图产物：`/llms.txt`、`/catalog.json` 真实生成为静态文件。
- 资产：复用 v1 `assets/`，按新品牌（黑 + 青绿）重做主题。
